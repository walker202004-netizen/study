# NotebookLM 提示词库

Google NotebookLM 是一个由AI驱动的研究助手，可以帮助用户快速从各种信息源中进行研究、学习和创建内容。本文档收集了针对 NotebookLM 优化的高质量提示词和使用技巧。

## 💡 NotebookLM 使用技巧

1. **上传多个来源** - 可以同时上传多个文档或网页，NotebookLM会进行综合分析
2. **细化问题** - 提问时要具体，这样会获得更精准的答案
3. **引用追踪** - NotebookLM会标注信息出处，便于验证
4. **迭代优化** - 根据初始结果继续提问和优化

## 🚀 快速开始

1. 访问 [NotebookLM](https://notebookllm.google.com/)
2. 创建新 Notebook
3. 上传相关文档或资源
4. 使用本文档中的提示词提问和分析
5. 根据需要调整和优化提示词

## 📝 提示词

### 构建专业知识库 - 资源搜寻与分类

```
我正在使用 NotebookLM 构建 [领域] 专家知识库。请帮我在 YouTube 上搜寻 8-10 个高质量视频，要求如下：

【覆盖三个维度】：
1. 基础架构：包含 [基础内容示例，如安装、核心概念、基本操作]
2. 高阶逻辑：包含 [进阶内容示例，如复杂功能、代码集成、错误处理]
3. 实战案例：包含 [实战应用示例，如具体项目、多系统集成、复杂工作流]

【筛选标准】：
1. 优先选择官方频道或公认专家的视频
2. 必须有清晰的口述讲解，避免纯录屏无解说
3. 寻找包含大量"逻辑解释"而非仅仅"鼠标操作"的视频
4. 如果同一个主题有多个优质视频，请优先选择讲解最深入、最清晰、逻辑链条最完整的那一个。

【输出格式】：
对于每个视频，请列出：
- 标题与链接（附带发布年月）
- 难度等级（初级/中级/专家）
- 关键节点/功能清单（列出该视频中重点演示的 3-5 个核心要素）
- 核心逻辑总结（描述该视频解决的核心问题）

【整体推荐与分层】：
- 优先级排序： 请按学习推荐程度对上述视频进行排序，并简要说明理由。
- 基础文档推荐： 明确指出哪 2-3 个视频最适合作为知识库的“初始锚点文档”（即信息最全、定义最准的视频）。
```

### 视频内容快速总结
```
作为我的高效知识管理助手，请针对我刚刚添加的视频源（Source），进行结构化的核心内容抽取。你的目标是让我用最少的时间看清视频全貌，并帮助我做出“是否需要亲自精看视频”的决策。

请严格按照以下模块输出，保持语言精炼、字字珠玑：

1. 🎯 一句话核心主旨
用最核心的一句话，说明这个视频到底解决了一个什么本质问题或传达了什么核心观点。

2. 🗺️ 5分钟鸟瞰（核心骨架与含金量评分）
请梳理出视频的 3-5 个核心阶段/逻辑板块。每个板块请用以下格式输出：
- [时间范围/章节名]：核心观点/关键结论（一句话）。【含金量评分：⭐/⭐⭐/⭐⭐⭐】（注：3星代表有全新见解、硬核干货或颠覆性观点；1星代表常识或老生常谈）。

3. 💎 核心亮点与“非听不可”的 Case/金句
视频中是否有让人眼前一亮的具体案例、实验数据、独特的方法论，或者让人警醒的金句？（如果没有，请直接写“无明显独特亮点”，不要编造）。

4. 💡 决策建议（我需要精看吗？）
请客观评估，并给出以下三种结论之一及原因：
- 【建议精看】：理由是视频中包含大量细节、实操步骤、或者高度依赖讲者的语气/图表演示，仅看摘要会遗漏 80% 的价值。
- 【建议速览/跳读】：理由是只有第 X 章节或 X 观点有新意，其余部分价值有限，建议直接去相应时间点拉进度条。
- 【无需精看】：理由是内容偏向科普、常识，或者你刚才的摘要已经完全覆盖了核心价值。
```

### 内容综合总结

```
请基于我上传的内容以及我们当前的对话，生成一份完整而凝练的总结，要求如下：

1. 保留所有关键概念、事件、人物、数据、结论和逻辑关系
2. 对话中出现的有价值问答或观点也要整合进总结
3. 不要省略任何信息，即使内容有重复也请保留
4. 输出清晰、按逻辑顺序排列，便于理解
5. 可以使用概览 → 详细内容的方式呈现

【输出结构】：
- 概览：核心主题和关键信息的高层概括
- 详细内容：按逻辑分块呈现所有重要信息
- 一句话总结：概括内容的核心主题或结论

请直接生成总结，不需要额外解释。
```
